## `BoundaryType` (enum)

```python
class BoundaryType(IntEnum):
    OPEN = 0           # ADCIRC code 0 — open ocean / external water
    MAINLAND = 1       # ADCIRC code 1 — mainland (no normal flux, no extra cols)
    ISLAND = 11        # ADCIRC code 11 — island boundary
    MAINLAND_FLUX = 20 # ADCIRC code 20 — mainland with normal-flux specified
    WALL = 1           # alias for MAINLAND (retained for backward compatibility with the existing enum)
```

**Notes**:
- `IntEnum` so the value compares equal to its ADCIRC numeric code: `BoundaryType.OPEN == 0` is `True`. Lets `fort14` writer emit codes by `int(bc)` without an explicit map.
- Codes outside this set are preserved on round-trip as plain `int` values in `BoundarySegment.bc_type`. The field is typed `BoundaryType | int`. A reader callsite that wants to switch on known codes uses `isinstance(seg.bc_type, BoundaryType)` to detect the named-vs-numeric case.
- `WALL` is a deliberate alias for `MAINLAND` because the existing code uses `WALL` for the same ADCIRC code; preserving the name avoids a backward-compat break.

## `BoundarySegment` (frozen dataclass)

```python
@dataclass(frozen=True, slots=True)
class BoundarySegment:
    node_ids: np.ndarray             # shape (N,), dtype int64; 0-based node indices in declaration order
    bc_type: BoundaryType | int      # named when known, raw int when reader saw an unmapped code
    is_open: bool                    # True iff this segment lives in the fort.14 "open boundary" block
```

**Invariants**:
- `node_ids.ndim == 1 and node_ids.dtype == np.int64`
- `0 <= node_ids[i] < mesh.n_nodes` for all `i`
- `is_open == (bc_type == BoundaryType.OPEN)` for v1 named codes; readers preserve `is_open` independently of `bc_type` to handle uncommon codes that ADCIRC places in either block
- Node ordering follows the declaration in the fort.14 source (or a deterministic order produced by the writer)

## `Mesh` (frozen dataclass)

```python
@dataclass(frozen=True, slots=True)
class Mesh:
    nodes: np.ndarray                # shape (N, 2), dtype float64 — (x, y) coordinates
    elements: np.ndarray             # shape (M, 3), dtype int64 — triangle vertex indices, 0-based
    boundaries: tuple[BoundarySegment, ...]   # ordered, immutable
    bathymetry: np.ndarray | None    # shape (N,), dtype float64 — elevation (positive-up); None when absent
    quality: np.ndarray | None       # shape (M,), dtype float64 — per-element quality metric; None until computed
    title: str = ""                  # AGRID line content from fort.14, or user-supplied label

    # Derived properties (not stored):
    @property
    def n_nodes(self) -> int: ...
    @property
    def n_elements(self) -> int: ...
    @property
    def n_boundaries(self) -> int: ...

    # Methods:
    def to_fort14(self, path: str | os.PathLike | TextIO) -> None: ...
    def plot(self, ax=None, **kwargs): ...      # optional[matplotlib]; raises ImportError if missing
    def __repr__(self) -> str: ...              # "Mesh(n_nodes=…, n_elements=…, min_q=…, …)"
    def __str__(self) -> str: ...               # multi-line human-readable summary
```

**Invariants**:
- `nodes.shape[0] == bathymetry.shape[0]` when `bathymetry is not None`
- `quality.shape[0] == elements.shape[0]` when `quality is not None`
- `elements` values are in `[0, n_nodes)`
- `boundaries` is a tuple (not a list) to enforce immutability in line with `frozen=True`

**Equality semantics**: `Mesh.__eq__` (auto-generated by `@dataclass`) compares NumPy arrays via `==`, then `.all()`. To support FR-014 round-trip equality, a `Mesh.equals(other, *, atol=1e-10, rtol=0)` helper accepts tolerances on coordinate and bathymetry comparisons (exact on connectivity).

## `Domain` (frozen dataclass)

```python
@dataclass(frozen=True, slots=True)
class Domain:
    sdf: Callable[[np.ndarray], np.ndarray]      # signed distance function: (N,2) -> (N,)
    bbox: tuple[float, float, float, float]      # (xmin, ymin, xmax, ymax)
    pfix: np.ndarray | None = None               # shape (K, 2), float64 — fixed points; None or empty if none
    pts: np.ndarray | None = None                # shape (P, 2), float64 — boundary discretization points; required by some stages
    bc_segments: tuple[BoundarySegment, ...] = ()    # optional pre-labeled boundary metadata for the output mesh
```

**Notes**:
- `sdf` is a callable to keep `Domain` independent of how the SDF was constructed (literal, polygon-derived, scalar-field-interpolated).
- `bbox` is a 4-tuple in `(xmin, ymin, xmax, ymax)` order to match Shapely / matplotlib conventions.
- `bc_segments` lets the user pre-declare which parts of the boundary are OPEN vs MAINLAND vs ISLAND. If empty, the pipeline applies a default labeling from `EnforceBoundaryConditions` (the existing faithful-port stage).

**Builder helpers** (not part of the dataclass — free functions in `admesh.api`):

```python
def domain_from_polygon(rings: list[np.ndarray], **kwargs) -> Domain: ...
def domain_from_sdf(sdf: Callable, bbox: tuple, **kwargs) -> Domain: ...
```

These are convenience constructors; users who want full control instantiate `Domain(...)` directly.

## `SizeField` (callable type alias + composer)

```python
SizeFieldFn = Callable[[np.ndarray], np.ndarray]
# Convention: input is (N, 2) array of (x, y) query points; output is (N,) array of target sizes.

def compose_size_field(
    builtins: list[SizeFieldFn],          # curvature, medial, bathymetry, tide, etc. — Phase 1
    user_contribs: list[SizeFieldFn] = (), # Phase 2
    combine: Callable[[list[np.ndarray]], np.ndarray] = np.minimum.reduce,
) -> SizeFieldFn:
    """Returns a callable that evaluates: combine([phase1_result, *evaluated_user_contribs])
    where phase1_result = np.minimum.reduce([f(pts) for f in builtins]).
    """
```

**Invariants**:
- Phase 1 always uses `np.minimum.reduce` regardless of `combine`. Constitution Principle I.
- `combine` is only applied between the Phase 1 result and the user contributions.
- All built-in callables MUST be the existing faithful-port functions wrapped to match `SizeFieldFn`. The wrappers MUST NOT alter inputs or outputs of the faithful-port functions.

## `Fort14File` (not a class — a module-level pair)

The fort.14 format does not need a wrapping class. The two functions are:

```python
def read_fort14(path: str | os.PathLike | TextIO) -> Mesh: ...
def write_fort14(mesh: Mesh, path: str | os.PathLike | TextIO) -> None: ...
```

`Mesh.to_fort14(path)` is a thin method that forwards to `write_fort14(self, path)`.

**Errors**:

```python
class Fort14ParseError(ValueError):
    """Raised by read_fort14 on malformed input. Always carries .line_no, .expected, .actual."""
    line_no: int
    expected: str
    actual: str
```

## State transitions

The data model is intentionally **stateless**. `Mesh`, `Domain`, and `BoundarySegment` are immutable. There are no entity lifecycles, no draft/published states, no pending-write states. `triangulate(domain) -> Mesh` is a pure function (modulo NumPy's RNG, which is seedable per FR-001).
